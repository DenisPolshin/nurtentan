# Контекст проекта (PROJECT_CONTEXT_FOR_AI)

## Обзор проекта
Веб-приложение для изучения управления немецких глаголов (Verben mit Präpositionen). 
**Основная цель:** Практика связок "Глагол + Предлог + Падеж" через интерактивные упражнения.
**База данных:** 740 уникальных предложений, охватывающих уровень A2-B1.

## Технологический стек
- **Framework:** Next.js (App Router), TypeScript, Tailwind CSS 4.
- **ORM & DB:** Prisma с SQLite (`dev.db`).
- **Auth:** NextAuth v5 (Auth.js) — Credentials + Google.
- **i18n:** `next-intl` для интерфейса + таблица `SentenceTranslation` для динамического контента.
- **UI:** Radix UI (Popover), Vaul (Drawer для мобильных "bottom sheets"), Framer Motion, Lucide React.
- **Переводы:** Скрипт на базе `google-translate-api-x` для пакетного перевода всех 740 предложений.

## Архитектура данных (Prisma)
- `Verb`: Глагол, предлог, падеж.
- `Sentence`: Шаблон предложения с плейсхолдерами `____`.
- `SentenceTranslation`: Переводы предложений на 7 языков (RU, TR, SR, SQ, UK, PL, HU).
- `UserProgress`: Отслеживание изученных глаголов (используется для логики уроков).

## Функционал и логика уроков
- **Алгоритм выбора:** Урок состоит из 10 вопросов. Логика: **8 новых** (неизученных) и **2 для повторения** (уже изученных).
- **Проверка ответа:** Сравнение с нормализацией (регистр, лишние пробелы, умлауты `ä -> ae`).
- **Выбор языка:** Кастомный `LanguagePicker`. При регистрации пользователь выбирает родной язык, который затем используется для отображения переводов в уроках.

## UI/UX Решения
- **LanguagePicker:** 
  - Десктоп: Выпадающий список (Radix Popover).
  - Мобилки: Нативный "bottom sheet" (Vaul Drawer).
  - Исправлено вертикальное выравнивание кодов языков и меток.
  - Использование `leading-snug` для предотвращения обрезания букв с нижними элементами (например, **ç** в Türkçe).
- **Регистрация:** Полностью на немецком языке ("Wähle deine Muttersprache").
- **Брендинг:** Удалены все упоминания "Duolingo-Stil", проект имеет собственный чистый стиль.

## Процесс деплоя (`deploy.bat`)
Автоматизированный скрипт для Windows, который:
1. Синхронизирует код через SSH.
2. Останавливает PM2.
3. Выполняет `prisma migrate reset` (полная очистка).
4. Запускает `seed-verbs.js` (генерация 740 предложений).
5. Запускает `translate-sentences.js` (перевод на все языки через Google API).
6. Собирает проект и перезапускает PM2.

## История проблем и решений (Сегодняшние правки)

### 1. Проблема: Дублирование приставок (напр. "vorvorbereiten")
- **Причина:** В логике генерации предложений с модальными глаголами приставка (`particle`) добавлялась к инфинитиву, который уже её содержал.
- **Решение:** Введен параметр `pureVerb` в скрипте сидинга, который содержит глагол без приставки для использования в сложных конструкциях.

### 2. Проблема: Пропажа переводов на сервере после деплоя
- **Причина:** Команда `seed` сбрасывала ID в базе, из-за чего связи в `SentenceTranslation` становились невалидными.
- **Решение:** Включен запуск `translate-sentences.js` в основной цикл деплоя сразу после сидинга.

### 3. Проблема: Ошибки грамматики в немецких текстах
- **Причина:** Неверное управление (Dativ/Akkusativ) для некоторых глаголов (напр. `sich sehnen nach` требует Dativ).
- **Решение:** Проведен полный аудит 740 предложений, исправлены падежи и капитализация существительных.

### 4. Проблема: Рекурсивная ошибка в мобильном Drawer
- **Причина:** Компонент `DrawerOverlay` в `ui/drawer.tsx` ошибочно вызывал сам себя.
- **Решение:** Исправлена ссылка на `DrawerPrimitive.Overlay`.

### 5. Проблема: Обрезание текста снизу в выборе языка
- **Причина:** Использование `leading-none` (line-height: 1) приводило к обрезанию нижних элементов букв (descenders).
- **Решение:** Заменено на `leading-snug` (line-height: 1.375) и удалены ручные коррекции `pt-[1px]`.

### 6. Проблема: Слом UI при автопереводе Google Translate
- **Причина:** Google Translate изменяет DOM в обход React, что приводит к падению приложения при обновлении стейта (NotFoundError: Failed to execute 'removeChild' on 'Node').
- **Решение:** В `layout.tsx` добавлен атрибут `translate="no"`, класс `notranslate` и мета-тег `google: 'notranslate'`.

### 7. Проблема: Скачки верстки и зум на мобильных
- **Причина:** Высота `100vh` не учитывает скрывающуюся строку адреса. iOS Safari зумит страницу, если шрифт в `<input>` меньше 16px.
- **Решение:** Заменено `vh` на `dvh` (Dynamic Viewport Height). Всем инпутам добавлен класс `text-base` (16px) для мобильных устройств. Добавлен `overflow-x-hidden` в `layout.tsx`.
