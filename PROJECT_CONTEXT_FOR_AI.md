# Контекст проекта (для другого ИИ)

## Цель продукта
Сайт для изучения немецких глаголов и их предлогов (глагол + предлог + падеж). Основной фокус — практика связок через короткие упражнения и мгновенную проверку.

## Стек и архитектура
- Next.js (App Router), проект расположен в `app/`
- TypeScript + React
- next-intl для локализации (сейчас используется только `de`)
- next-auth (Credentials + Google) + Prisma Adapter
- Prisma + SQLite
- UI: shadcn/ui компоненты + Tailwind

Ключевые каталоги:
- `app/src/app` — страницы и API роуты App Router
- `app/src/app/[locale]` — локализованные роуты (`/de/...`)
- `app/src/proxy.ts` — “middleware/proxy” для i18n + auth
- `app/prisma/schema.prisma` — схема БД
- `app/scripts/seed-verbs.js` — сидинг глаголов/предложений

## Маршруты (важные)
- `/de/login` — логин/регистрация (клиентская страница)
- `/de` — дашборд (серверная страница, требует сессию)
- `/de/lesson?type=akkusativ|dativ` — урок (серверная + клиентская часть)
- `/api/auth/[...nextauth]` — next-auth handlers
- `/api/auth/register` — регистрация пользователя (Credentials)

## Локализация (next-intl)
- Конфиг локалей: `app/src/i18n/routing.ts` (сейчас `locales: ['de']`, `defaultLocale: 'de'`)
- Сообщения: `app/src/locales/de/translation.json`
- Важно: без корректного `proxy.ts` прямой заход на `/login` может не переписываться в `/de/login`

## Авторизация (next-auth)
- Конфиг: `app/src/auth.config.ts`
  - Credentials: email + password (bcrypt)
  - Google provider (через env)
  - `pages.signIn = '/login'`
  - `authorized` callback: публичные роуты — `/login`, `/de/login`, `/api/auth/*`, `/_next/*`, `/favicon.ico`
  - Локаль в логике приведена к `de`
- Обёртка: `app/src/auth.ts` (PrismaAdapter, JWT-сессии)

## “Middleware/Proxy”
- `app/src/proxy.ts`:
  - Оборачивает запросы `auth(...)`
  - Пропускает API и статику
  - Применяет next-intl middleware (переписывает/редиректит локаль)
  - matcher: `'/((?!api|_next|.*\\..*).*)'`

## База данных (Prisma)
Файл: `app/prisma/schema.prisma`

Основные модели:
- `User`, `Account`, `Session` — next-auth
- `Verb`:
  - `infinitive` (строка для отображения, иногда включает “sich …” или “es …”)
  - `preposition` (предлог)
  - `case` (`Akkusativ` или `Dativ`)
  - `translation` (опционально, сейчас в сидере ставится `null`)
- `Sentence`:
  - `text` — предложение-шаблон с ровно 2 плейсхолдерами `____` (1-й для глагола, 2-й для предлога)
  - `verbAnswer` — правильный ответ для 1-го плейсхолдера
  - `prepAnswer` — правильный ответ для 2-го плейсхолдера (должен совпадать с `Verb.preposition`)
  - `translation` — сейчас используется как “подсказка после ответа”: заполненное корректное предложение на немецком (без `____`)
- `UserProgress` — сейчас не обновляется в процессе урока (дашборд читает `isLearned`, но запись прогресса пока не реализована)

## Как работает урок
### Серверная часть
Файл: `app/src/app/[locale]/(main)/lesson/page.tsx`
- Проверяет сессию через `auth()`, иначе `redirect('/login')`
- Берёт `type` из query (`akkusativ` / `dativ`) и делает `prisma.verb.findMany(where: { case: ... }, include: { sentences: true })`
- “Разворачивает” `Verb.sentences` в массив `questions` и берёт 10 случайных
- Собирает список предлогов из `verbs.map(v => v.preposition)` и передаёт в клиент

### Клиентская часть
Файл: `app/src/app/[locale]/(main)/lesson/lesson-client.tsx`
- Рендерит предложение, вставляя 2 поля ввода в места `____`
- Проверка ответа:
  - сравнение нормализованных строк (trim/lowercase, пробелы, ß→ss, ä→ae, ö→oe, ü→ue, удаление лишних символов)
- После проверки показывает:
  - статус “Richtig/Falsch”
  - правильный ответ (глагол + предлог)
  - строку-подсказку `Sentence.translation` (появляется только после ответа)

## Регистрация
Файлы:
- Клиент: `app/src/app/[locale]/login/page.tsx`
- API: `app/src/app/api/auth/register/route.ts`

Логика:
- `POST /api/auth/register` создаёт пользователя (bcrypt)
- Если email уже существует — отдаёт `{ error: 'email_exists' }` и статус 400
- UI на экране показывает более понятную ошибку для `email_exists`

## Сидинг глаголов и заданий
Файл: `app/scripts/seed-verbs.js`
- Сидит 185 связок (Akkusativ + Dativ), по 1 заданию на глагол
- Каждый `Sentence.text` содержит ровно 2 `____`
- Для рефлексивных глаголов по умолчанию добавляется `uns` в предложение
- Для некоторых глаголов есть “custom” шаблоны (отделяемые приставки, фиксированные части, корректные подлежащие и т.д.)
- `Sentence.translation` записывается как полностью заполненное корректное предложение (на немецком)
- По умолчанию:
  - очищаются `UserProgress`, `Sentence`, `Verb`
  - пользователи НЕ очищаются
- При запуске с флагом `--reset-users` дополнительно очищаются `Session`, `Account`, `User`

Команды:
- `node scripts/seed-verbs.js` — пересоздать только глаголы/предложения и прогресс
- `node scripts/seed-verbs.js --reset-users` — дополнительно обнулить пользователей

## Аудит данных
Файл: `app/scripts/audit-verbs.js`
- Проверяет:
  - дубликаты по (infinitive, preposition, case)
  - что в `Sentence.text` ровно 2 плейсхолдера `____`
  - что `Sentence.prepAnswer === Verb.preposition`
  - что нет кириллицы в текстах заданий/ответах/подсказках
  - что `Sentence.translation` уже заполнено и не содержит `____`

## Текущие известные ограничения
- Прогресс (`UserProgress`) не обновляется по результатам урока: сейчас счёт хранится только на клиенте, а в БД не записывается.
- Данные по глаголам формируются из сидера; качество/естественность примеров зависит от настроек `custom` и словаря объектов в `seed-verbs.js`.
