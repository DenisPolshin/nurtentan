# Промпт для ИИ: Интеграция перевода предложений через DeepL и выбор родного языка

**Контекст проекта:**
Проект на Next.js (App Router), TypeScript, Prisma (SQLite), NextAuth v5, Tailwind CSS, shadcn/ui.
Суть приложения: изучение немецких глаголов с предлогами (уровень А2-B1). В базе есть модель `Sentence`, в которой хранятся предложения с пропусками (поле `text`, например: "Ich ____ mich ____ das Wetter."). Сейчас поле `translation` в модели `Sentence` содержит правильно заполненное немецкое предложение, которое используется как подсказка после ответа.

**Задача:**
Необходимо добавить функционал выбора родного языка пользователя при регистрации и автоматический перевод заданий на этот язык через API DeepL. Перевод должен отображаться над немецким предложением во время урока.

Пожалуйста, реализуй эту задачу по следующим шагам.

### Шаг 1. Обновление схемы базы данных (Prisma)
1. В модель `User` добавь поле `nativeLanguage String? @default("RU")` (или без дефолта, если требуется обязательный выбор).
2. Создай новую модель `SentenceTranslation` для кэширования переводов, чтобы не дергать DeepL каждый раз для одних и тех же предложений:
```prisma
model SentenceTranslation {
  id         String   @id @default(cuid())
  sentenceId String
  language   String   // Например, "RU", "EN", "ES", "TR"
  text       String   // Переведенный текст
  sentence   Sentence @relation(fields: [sentenceId], references: [id], onDelete: Cascade)

  @@unique([sentenceId, language])
}
```
*(Не забудь добавить `translations SentenceTranslation[]` в модель `Sentence` и запустить `npx prisma db push`)*.

### Шаг 2. Обновление авторизации (NextAuth)
1. Обнови форму регистрации (`app/src/app/[locale]/login/page.tsx` и `app/src/app/api/auth/register/route.ts`), добавив поле `Select` (из shadcn/ui) для выбора родного языка (например: Русский, Английский, Турецкий, Испанский, Арабский и т.д.). Сохраняй это значение при создании пользователя.
2. Обнови конфигурацию NextAuth (`app/src/auth.ts` и `auth.config.ts`), чтобы поле `nativeLanguage` добавлялось в сессию (`session.user.nativeLanguage`) и в JWT токен. Это позволит легко получать язык текущего пользователя на серверных компонентах.
3. *Опционально:* Если пользователь авторизуется через Google, его `nativeLanguage` будет `null`. В `proxy.ts` или в `layout.tsx` (или в самом `lesson/page.tsx`) сделай проверку: если у авторизованного пользователя `nativeLanguage === null`, редиректить его на страницу онбординга `/onboarding` (которую нужно создать), где он выберет свой язык.

### Шаг 3. Интеграция DeepL (Серверная утилита)
1. Создай файл `app/src/lib/deepl.ts`.
2. Напиши функцию `translateSentence(text: string, targetLang: string)`, которая будет обращаться к бесплатному API DeepL (`https://api-free.deepl.com/v2/translate`).
3. Ключ API должен браться из переменных окружения: `process.env.DEEPL_API_KEY`.
4. Обрати внимание, что переводить нужно полностью заполненное немецкое предложение (которое сейчас хранится в `Sentence.translation`), а не предложение с пропусками `____`.

### Шаг 4. Обновление логики урока (`lesson/page.tsx`)
В серверном компоненте `LessonPage`:
1. Получи `nativeLanguage` пользователя из сессии.
2. Для выбранных 10 случайных предложений сделай проверку в базе данных (`SentenceTranslation`), есть ли для них уже перевод на этот язык.
3. Если перевода нет — вызови утилиту `translateSentence`, получи перевод, сохрани его в `SentenceTranslation` и верни вместе с вопросом.
4. Добавь новое поле `nativeTranslation` в объект вопроса (тип `Question`), который передается в клиентский компонент `LessonClient`.

### Шаг 5. Обновление UI клиента (`lesson-client.tsx`)
1. В интерфейсе `Question` добавь `nativeTranslation: string | null`.
2. В самом компоненте `LessonClient`, над блоком с заданием (где отображаются части предложения с инпутами), выведи перевод на родной язык.
3. Оформи его визуально так, чтобы он выглядел как подсказка-контекст (например, светло-серым цветом, курсивом, с иконкой 🌐 или 💡), чтобы пользователь сразу понимал смысл того, что он сейчас будет заполнять.

**Ограничения и требования:**
- Ничего не ломай в текущей логике проверки ответов.
- Используй компоненты shadcn/ui для селекта языка и возможных новых форм.
- Строго соблюдай существующую архитектуру локализации (next-intl).
- Не используй `npm install` без явной необходимости (REST-запрос к DeepL можно сделать через нативный `fetch`).
